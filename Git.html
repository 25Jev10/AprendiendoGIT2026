Git es un sistema de control de versiones que funciona en tu computadora, mientras que GitHub es una plataforma en la nube que usa Git para permitir colaboración y almacenamiento de proyectos. En pocas palabras: Git es la herramienta, GitHub es el servicio que la aprovecha.

🔹 ¿Qué es Git?

  • Definición: Git es un software de control de versiones distribuido creado por Linus Torvalds en 2005.

  • Función principal: Permite llevar un historial de cambios en el código, volver a versiones anteriores y trabajar en paralelo con ramas.

  • Uso típico: Se instala en tu PC y se usa con comandos (git init, git commit, git branch, etc.).

  • Ventaja clave: No depende de internet; puedes trabajar localmente y sincronizar después.

🔹 ¿Qué es GitHub?

  • Definición: GitHub es una plataforma en línea (propiedad de Microsoft) que aloja repositorios Git en la nube.

  • Función principal: Facilita compartir proyectos, colaborar con otros desarrolladores y gestionar código de manera social.

  • Características adicionales:

  • Pull requests (propuestas de cambios).

  • Issues (gestión de tareas/errores).

  • Integración con CI/CD (automatización).

  • Interfaz gráfica web para visualizar repositorios.

🔹 Verificar configuración

Puedes comprobar lo que guardaste con:

Esto mostrará tu nombre, correo y otras configuraciones.

🔹 Diferencia entre --global y sin él

  • --global: se aplica a todos los repositorios en tu computadora.

  • Sin --global: se aplica solo al repositorio actual. Ejemplo:

git config user.name "Luis ProyectoX"

git config user.email "proyectox@example.com"

👉 Con esto ya tienes tu identidad configurada y cada commit quedará firmado con tu nombre y correo.

🔹 1. git init

  • Función: Inicializa un nuevo repositorio Git en una carpeta.

  • Uso:

git init

Esto crea una carpeta oculta en la pc .git donde se guarda todo el historial de versiones.

🔹 2. git clone

  • Función: Copia un repositorio existente (local o remoto, por ejemplo en GitHub) a tu máquina.

  • Uso:

git clone https://github.com/usuario/proyecto.git

Descarga el proyecto completo con su historial.

🔹 3. git add

  • Función: Añade archivos al área de preparación (staging area) para incluirlos en el próximo commit.

  • Uso:

git add archivo.txt

O para todos los archivos modificados:

git add .

🔹 4. git commit

  • Función: Guarda los cambios en el historial del repositorio con un mensaje descriptivo.

  • Uso:

git commit -m "Agrego función de login"

Cada commit es como una “foto” del estado del proyecto.

🔹 5. git push

  • Función: Envía tus commits locales al repositorio remoto (por ejemplo, GitHub).

  • Uso:

git push origin main

Aquí origin es el nombre del remoto y main la rama principal.

🔹 Flujo básico con estos comandos

  1. Crear o clonar un repositorio (git init o git clone).

  2. Modificar archivos.

  3. Preparar cambios (git add).

  4. Guardar cambios (git commit).

  5. Subirlos al servidor remoto (git push).
